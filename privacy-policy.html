Privacy Policy

Application: Threads Social Listening Tool（internal use）
Operator: 寰居有限公司 (Huanju Co., Ltd.)
Contact e-mail: junhao8651@gmail.com
Effective date: 2026-06-15

This application is an internal social-listening tool operated by the party named above for its own small e-commerce business. It uses the official Threads API to find public Threads posts that mention a small set of brand- or product-related keywords, so that a human team member can review them and decide whether to reply manually. This policy explains what data the tool accesses, how it is used, stored, and deleted.

1. What data we access

Through the official Threads API (with the user's authorization and Meta-approved permissions threads_basic and threads_keyword_search), the tool reads only public Threads posts that match the keywords we monitor. For each matching post we may store:

  • Post ID
  • Post text
  • Author username
  • Permalink (public URL of the post)
  • Timestamp

We do not collect private messages, e-mail addresses, phone numbers, passwords, payment information, or any non-public personal data.

2. How we use the data

  • De-duplicate results and sort them by time.
  • Track the daily volume of each keyword to detect early rising trends.
  • Produce an internal "to-review" list so a human can read the posts.
  • Send a notification to the operator (via LINE, e-mail, or Telegram) when a keyword shows an unusual rise in volume.

Any reply to a post is written and sent manually by a human after review. The tool never auto-replies, never auto-posts, never sends bulk messages, and never operates multiple accounts.

3. What we do NOT do

  • We do not sell, rent, or share the data with third parties for advertising.
  • We do not use the data to build profiles of individuals beyond the public post content needed for the tasks above.
  • We do not perform any automated posting, mass outreach, or evasion of platform protections.

4. Storage, retention, and security

Data is stored in a local database (SQLite) on the operator's own computer. It is not uploaded to any public server. The data is retained only as long as needed for the review-and-reply workflow, and may be deleted by the operator at any time. Access to the computer and the database is limited to the operator.

5. Third-party services

ServicePurpose
Meta / Threads APISource of the public posts
LINE Messaging API / Telegram / e-mail (SMTP)Sending trend notifications to the operator only

We use these services only to operate the tool as described above.

6. Data deletion

Because the data is stored locally, it can be removed by deleting the local database file. If you are a Threads user and want any stored copy of your public post removed from our local database, contact us at the e-mail above and we will delete it.

7. Changes to this policy

We may update this policy from time to time. The latest version will always be available at this URL.

8. Contact

Questions about this policy: junhao8651@gmail.com

本工具是上述營運者為自家小型電商所使用的內部社群聆聽工具。它透過官方 Threads API， 找出提及我們所監控的少數品牌／產品相關關鍵字的公開 Threads 貼文， 供真人團隊審閱後決定是否親自回覆。以下說明本工具會存取哪些資料、如何使用、儲存與刪除。

1. 我們存取哪些資料

透過官方 Threads API（在使用者授權、且取得 Meta 核准的 threads_basic 與 threads_keyword_search 權限下），本工具只讀取符合監控關鍵字的公開貼文。 每則貼文可能儲存：貼文 ID、貼文內容、作者帳號、貼文公開連結、發文時間。

我們不會蒐集私訊、Email、電話、密碼、付款資訊或任何非公開的個人資料。

2. 我們如何使用這些資料

  • 去除重複、依時間排序。
  • 追蹤每個關鍵字的每日聲量，偵測早期上升趨勢。
  • 產生內部「待審閱清單」供真人閱讀。
  • 當某關鍵字聲量異常上升時，透過 LINE／Email／Telegram 通知營運者本人。

所有回覆一律由真人審閱後親自撰寫並送出。本工具不自動回覆、不自動發文、 不大量發送訊息、不操作多個帳號。

3. 我們不會做的事

  • 不販售、出租或為廣告目的把資料分享給第三方。
  • 不超出上述任務所需，對個人建立額外側寫。
  • 不進行任何自動發文、大量觸及或規避平台防護的行為。

4. 儲存、保留與安全

資料儲存在營運者自己電腦上的本機資料庫（SQLite），不會上傳到任何公開伺服器。 資料僅在審閱與回覆流程所需期間保留，營運者可隨時刪除。資料庫的存取僅限營運者本人。
